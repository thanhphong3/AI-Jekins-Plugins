Bước 1: Kích Hoạt Tiến Trình (Triggering)
Listener & Actions
  • Chạy tự động (Auto Trigger): Khi build hoàn thành, AiLogAnalyzerRunListener đón sự kiện và kiểm tra trạng thái build. Nếu kết quả là Result.FAILURE hoặc Result.UNSTABLE, nó sẽ khởi tạo background thread.
  • Chạy thủ công (Manual Trigger): Người dùng click nút "Analyze build log" trên giao diện Jenkins của build. Yêu cầu POST được gửi đến endpoint doStartAnalysis trong AiLogAnalyzerAction để kích hoạt.
  • Khởi tạo luồng xử lý riêng biệt (Background Thread) để tránh gây nghẽn luồng chính của Jenkins.
Bước 2: Lọc Log Thông Minh (Log Filtering)
Token Optimization
  • Quét lỗi đặc trưng: Sử dụng Regex để quét các từ khóa/mẫu lỗi phổ biến (Unity, Xcode, C# compile errors, Code signing, clang errors).
  • Giữ lại ngữ cảnh: Giữ lại dòng chứa lỗi và lấy thêm 3 dòng phía trước cùng 5 dòng phía sau dòng lỗi đó.
  • Loại bỏ nhiễu: Bỏ qua các dòng log thông thường không có tín hiệu lỗi, giảm dung lượng log gửi đi gấp 5-10 lần (tiết kiệm token của LLM).
  • Dự phòng: Nếu không tìm thấy lỗi, plugin sẽ quét cảnh báo (Warning). Nếu không có cảnh báo nào, nó sẽ fallback lấy 500 dòng log cuối cùng.
Bước 3: Lựa Chọn Model AI (Model Negotiation)
Autodetect & Fallback
  • Tự động phát hiện (Autodetect): Gửi yêu cầu GET đến API endpoint /models hoặc /v1/models để lấy danh sách các model đang hoạt động trên Server.
  • Xử lý lỗi (Fallback list): Nếu model được chỉ định chạy lỗi (ví dụ do server overload), plugin tự động chuyển sang thử các model dự phòng khác trong danh sách (ví dụ: chuyển từ gemini-1.5-pro sang gemini-1.5-flash).
Bước 4: Đóng Gói Và Gửi API Payload
API Request
  • Tạo prompt tối ưu: Tích hợp Prompt chỉ dẫn định dạng Markdown cụ thể (Tóm tắt build dạng bảng, Phân tích nguyên nhân gốc rễ, Đề xuất bước khắc phục).
  • Payload JSON chuẩn: Đóng gói payload JSON theo định dạng OpenAI/Gemini Chat Completions (messages array chứa role user và content).
  • Gửi HTTPS: Thực hiện HTTP POST, truyền token bảo mật (Authorization Bearer / API Key) và thiết lập timeout 30 giây.
Bước 5: Hiển Thị & Dịch Kết Quả
Jenkins UI & Translator
  • Injected UI: Dữ liệu kết quả phân tích (Markdown) được lưu trữ trực tiếp vào build meta-data và hiển thị đẹp mắt tại trang tóm tắt build (Build Summary).
  • Dịch tiếng Việt (Translation): Khi người dùng click nút "Xem Tiếng Việt", plugin gọi endpoint dịch thuật. AI sẽ dịch nội dung phân tích sang tiếng Việt mà không làm hỏng cấu trúc Markdown, bảng biểu hay dịch sai các đoạn log code thô.
Trực Quan Hóa Bộ Lọc Log (LogFilter Simulator)

Nhấn nút bên dưới để mô phỏng thuật toán rút gọn log của LogFilter.java. Dòng lỗi màu đỏ và ngữ cảnh màu vàng sẽ được giữ lại, các dòng rác màu xám sẽ bị lược bỏ.

Raw Console Log (Log Thô từ Jenkins) 30 lines
Loading log...
Filtered Log (Gửi lên AI API) Chưa lọc
Bấm "Chạy Bộ Lọc Log" để xem kết quả rút gọn...
Cấu Trúc Yêu Cầu Gửi Cho AI (API Payload JSON)

Payload JSON tiêu chuẩn được gửi lên endpoint của AI Server chứa prompt và dữ liệu log đã rút gọn. 

{
  "model": "gemini-1.5-pro",
  "prompt": "You are a DevOps and Build Engineer expert. Analyze the build log...",
  "log": "[AI Log Analyzer: Pre-filtered high-signal build log...]",
  "messages": [
    {
      "role": "user",
      "content": "You are a DevOps and Build Engineer expert... \n\n [Filtered Log Content]"
    }
  ]
}
